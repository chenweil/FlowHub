# Flow Hub v1.0.0 发布清单

## 版本定位

- 版本号：`1.0.0`
- 发布分支：`release/qwen-1.0.0`
- 核心主题：从 iFlow 驱动切换为 Qwen 驱动的首个正式版本

## 发布说明摘要

- Flow Hub 现在默认通过 Qwen CLI + ACP stdio 驱动 Agent。
- 已支持 Qwen 历史会话导入、工作区文件列表查看、长任务活动状态提示等关键能力。
- 当前保留 `com.iflow.workspace` 标识与若干 `iflow-*` 本地存储 key，用于兼容已有安装与升级数据。

## 发布前检查

1. 确认当前分支为 `release/qwen-1.0.0`。
2. 确认 `package.json`、`src-tauri/Cargo.toml`、`src-tauri/tauri.conf.json` 版本均为 `1.0.0`。
3. 运行验证命令：
   - `npm test -- --run`
   - `npm run build`
   - `cd src-tauri && cargo test`
4. 手工验证：
   - 能成功连接 Qwen Agent；
   - 能发送消息并正常收到流式响应；
   - “更多 -> 工具”可显示活动摘要；
   - 工作区文件列表面板可打开；
   - HTML 预览可正常打开本地生成文件。
5. 确认 `CHANGELOG.md` 已包含 `1.0.0` 条目。

## 打 tag 与出包顺序

1. 提交当前发布内容。
2. 创建发布 tag：
   - `git tag -a v1.0.0 -m "Release v1.0.0"`
3. 构建安装包：
   - `npm run tauri:build`
4. 检查产物目录：
   - `src-tauri/target/release/bundle/`
5. 验证安装包名称、版本号、启动行为与图标。
6. 推送分支与 tag：
   - `git push origin release/qwen-1.0.0`
   - `git push origin v1.0.0`

## 当前已知注意事项

- 当前工作区仍有未提交改动，因此不能把 `v1.0.0` tag 直接打在工作区状态上；必须先提交。
- `bundle identifier` 当前仍为 `com.iflow.workspace`，这是有意保留，用于保证已有安装升级路径稳定。
- 若 macOS 出包阶段触发签名或系统权限问题，需要根据本机证书配置补做签名检查。
