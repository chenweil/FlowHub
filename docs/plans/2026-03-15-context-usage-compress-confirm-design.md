# 上下文压缩入口调整设计文档

| 项目 | 内容 |
|---|---|
| 版本 | v1.0 |
| 日期 | 2026-03-15 |
| 状态 | 设计已确认 |

## 1. 需求概述

| 项目 | 内容 |
|---|---|
| 用户目标 | 上下文使用进度条整块可点击触发压缩确认；移除旁边单独压缩按钮 |
| 关键约束 | 不支持压缩时不可点击；仅对话过的会话可压缩；首次连接后选中的会话不可压缩 |
| 成功标准 | 点击整块区域弹出“是否压缩上下文?”确认；不可压缩状态下无点击反馈；独立压缩按钮消失 |

## 2. 交互与UI

| 项目 | 设计 |
|---|---|
| 可点击区域 | 上下文使用条整块区域可点击 |
| 确认文案 | `是否压缩上下文?` |
| 禁用态 | 不可压缩时区域禁用且无 hover/点击反馈；显示为禁用视觉态 |
| 入口调整 | 移除独立压缩按钮与其样式 |

## 3. 逻辑与状态

| 项目 | 设计 |
|---|---|
| 可压缩判定 | 复用现有会话是否有对话内容的判断；并结合当前 Agent 连接状态与忙碌状态的既有校验 |
| 点击流程 | 可压缩时点击区域弹出确认；确认后触发既有 `/compress` 发送流程 |
| 状态刷新 | 沿用现有“上下文使用条更新”时机同步更新可点击与禁用态 |
| 环境差异 | dev/test/prod 共用同一判定逻辑，不引入 mock 或假数据 |

## 4. 错误与边界处理

| 场景 | 处理 |
|---|---|
| Agent 离线或忙碌 | 禁用点击；必要时复用既有错误提示逻辑 |
| 确认取消 | 不触发压缩 |
| 压缩失败 | 复用既有错误提示与状态恢复逻辑 |

## 5. 测试策略

| 类型 | 覆盖 |
|---|---|
| 交互测试 | 有对话内容时点击整块区域弹出确认；确认后触发压缩；取消不触发 |
| 禁用测试 | 空会话或首次连接会话不可点击；Agent 离线不可点击 |
| 回归测试 | 上下文使用条显示与颜色逻辑保持不变 |

## 6. 影响范围

| 文件 | 变更 |
|---|---|
| `index.html` | 移除压缩按钮 DOM |
| `src/dom.ts` | 移除压缩按钮引用 |
| `src/styles.css` | 移除压缩按钮样式；新增/调整进度条禁用态样式 |
| `src/features/app.ts` | 移除按钮事件绑定；在进度条点击中加入确认与可压缩判定 |
| `src/features/contextUsage.ts` | 可压缩状态驱动 UI 禁用态更新（复用现有更新时机） |

## 7. 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| 判定条件不一致导致可点击但不可压缩 | 中 | 复用现有压缩前校验逻辑作为单一来源 |
| 入口变化影响用户习惯 | 低 | 仅移动入口到进度条整块区域并保留提示文案 |
