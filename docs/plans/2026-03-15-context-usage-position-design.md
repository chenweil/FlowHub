# 上下文使用条位置调整设计文档

| 项目 | 内容 |
|---|---|
| 版本 | v1.0 |
| 日期 | 2026-03-15 |
| 状态 | 设计已确认 |

## 1. 需求概述

| 项目 | 内容 |
|---|---|
| 用户目标 | 将上下文使用条移动到输入提示行右侧，位于输入框下方 |
| 对齐规则 | 与发送按钮右边缘对齐后左移 8px |
| 约束 | 不改变功能逻辑，仅调整布局与样式 |

## 2. 交互与布局

| 项目 | 设计 |
|---|---|
| 位置 | 与 `input-hints` 同一行的右侧 |
| 行内布局 | `input-hints` 改为 `display:flex`，提示文字在左，上下文条在右 |
| 对齐 | 上下文条使用 `margin-left:auto`，并加 `margin-right: 8px` 形成对齐偏移 |
| 显隐 | 继续使用现有显示逻辑（无对话隐藏） |

## 3. 实施影响

| 文件 | 变更 |
|---|---|
| `index.html` | 将 `context-usage-wrapper` 移到 `input-hints` 内部 |
| `src/styles.css` | 调整 `.input-hints` 布局与 `.context-usage-wrapper` 边距 |

## 4. 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| 提示文本与上下文条挤压 | 低 | 使用 flex 与 auto 对齐；必要时允许提示文本换行 |
| 移动端布局变化 | 低 | 保持现有 `gap` 与字体尺寸，避免绝对定位 |

## 5. 验证要点

| 项目 | 说明 |
|---|---|
| 位置 | 上下文条位于输入提示行右侧 |
| 对齐 | 右边缘相对发送按钮右边缘左移约 8px |
| 功能 | 显隐与点击逻辑不变 |
